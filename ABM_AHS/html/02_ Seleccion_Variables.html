2. Feature Engineering y Selección de Variables¶

En este segundo notebook, llevaremos a cabo Feature Engineering que es el proceso de seleccionar, transformar o crear variables en un conjunto de datos con el objetivo de mejorar el rendimiento y la eficacia de los modelos.

Y tambin realizaremos la Seleccin de Variables que implica identificar y retener nicamente las variables ms relevantes y significativas para el problema en cuestin, descartando aquellas que pueden introducir ruido o tener un impacto limitado en la capacidad predictiva del modelo. La seleccin de variables busca simplificar y optimizar el modelo, mejorar la interpretacin de los resultados y reducir el riesgo de overfitting.

Feature Engineering¶

Sustitución de valores nulos (NaN)¶

In [5]:
# Initialize an empty list to store continuous variables
list_continuous_variable = []

# Iterate through each column in the 'pd_fraud' DataFrame
for column in pd_fraud.columns:
    # Check if the data type of the column is not 'category'
    if pd_fraud[column].dtype != 'category':
        # If not a categorical variable, add the column to the list of continuous variables
        list_continuous_variable.append(column)

Llevamos a cabo la sustitucin de valores nulos por la mediana, decidimos sustituirlos por la mediana ya que es una medida ms robusta que la media de manera que no se ven tan afectada por los valores extremos.

In [6]:
# Fill missing values in continuous variables with their respective column means
pd_fraud[list_continuous_variable] = pd_fraud[list_continuous_variable].apply(lambda x: x.fillna(x.mean()))

Llevamos a cabo la codificacin de las variables que contienene datos de tipo string para convertirlas en valores numricos.

En primer lugar, dado el anlisis exploratorio conocemos que existe una variable categrica, source, que nicamente tiene dos valores nicos, por tanto simplemente sustituimos la columna actual por una nueva columna que asigna a cada valor nico un 0 o un 1 segn corresponda.

Posteriomente llevamos a cabo la codificacin del resto de variables, estas variables presentan un mayor nmero de valores nicos pero no nos encontramos ante un gran nmero de ellos, es por eso por lo que decidimos aplicar la tcnica de one-hot encoding.

Escalado de variables¶

El escalado de variables es una etapa crucial diseada para asegurar que las caractersticas del conjunto de datos estn en una escala uniforme. Esta uniformidad es esencial porque muchos algoritmos son sensibles a las diferencias en la magnitud de las variables. Sin el escalado adecuado, las caractersticas con unidades ms grandes o rangos ms amplios pueden dominar la contribucin al modelo, sesgando los resultados y afectando la capacidad del modelo para aprender patrones de manera equitativa.

In [12]:
# Initialize a StandardScaler for feature scaling
scaler = StandardScaler()

# Separate the target variable and features from the DataFrame
target_variable = pd_fraud['fraud_bool']
features_to_scale = pd_fraud.drop(['fraud_bool'], axis=1)

# Scale the features using the StandardScaler
scaled_features = scaler.fit_transform(features_to_scale)

# Create a new DataFrame with the scaled features
scaled_df = pd.DataFrame(scaled_features, columns=features_to_scale.columns)

# Insert the target variable back into the DataFrame as the first column
scaled_df.insert(0, 'fraud_bool', target_variable)

# Update the original DataFrame 'pd_fraud' with the scaled DataFrame
pd_fraud = scaled_df

Eliminación de variables¶

Optamos por eliminar las siguientes variables, antes de entrenar el modelo.

  • device_fraud_count: Es una constante y mantenerla no aporta nada al modelo.

  • session_lenght_in_minutes y prev_address_months_count: Son dos variables con una muy escasa importancia como hemos visto a travs del algoritmo Lasso, adems la variable prev_address_months_count presenta un muy elevado nmero de nulos.

  • velocity_4w: Es una variable que posee poco importancia y tambin tiene una elevada correlacin negativa, superior al 80% con la variable month.
